# Система блогов с комментариями и лайками

## Введение

В этой лабораторной работе мы создадим полноценную систему блогов
с возможностью добавления постов, комментирования (включая вложенные
комментарии) и системой лайков. Мы изучим ключевые концепции Laravel:
миграции, Eloquent ORM, валидацию через Form Requests, Blade-шаблоны,
слоистую архитектуру, middleware и многое другое.

**Цели работы:**

- Научиться создавать сложные связи между моделями
- Освоить полиморфные отношения в Eloquent
- Понять принципы слоистой архитектуры и SOLID
- Научиться правильно организовывать бизнес-логику
- Освоить валидацию через Form Requests
- Изучить работу с middleware и фасадами Laravel

## Часть 2. Миграции — версионирование структуры базы данных

### Шаг 2.1. Концепция миграций в Laravel

**Миграции** — это система контроля версий для вашей базы данных.
Они позволяют:

- Определять структуру БД в PHP-коде
- Создавать таблицы и изменять их программно
- Откатывать изменения при необходимости
- Синхронизировать структуру БД между разработчиками

**Основные методы миграций:**

- `up()` — применение изменений (создание/изменение таблиц)
- `down()` — откат изменений (для отмены миграции)

### Шаг 2.2. Модификация таблицы users

Laravel уже содержит базовую миграцию для пользователей. Нам нужно
модифицировать её под нашу структуру.

Откройте файл `database/migrations/0001_01_01_000000_create_users_table.php`
и модифицируйте метод `up()`:

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id(); // BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
            $table->string('lastname', 50);
            $table->string('firstname', 50);
            $table->string('patronymic', 50)->nullable();
            $table->string('email', 100)->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken(); // Для функции "Запомнить меня"
            $table->timestamps(); // created_at и updated_at

            // Составной индекс для быстрого поиска по имени
            $table->index(['lastname', 'firstname'], 'idx_lastname_firstname');
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('users');
    }
};
```

**Объяснение конструкций Blueprint:**

- `id()` — первичный ключ типа `BIGINT UNSIGNED` с `AUTO_INCREMENT`
- `string('name', length)` — `VARCHAR(length)`
- `nullable()` — позволяет хранить `NULL`
- `unique()` — создаёт уникальный индекс
- `timestamps()` — добавляет `created_at` и `updated_at` типа `TIMESTAMP`
- `index([...], 'name')` — обычный индекс для ускорения запросов

### Шаг 2.3. Создание миграции для таблицы posts

Создайте новую миграцию:

```bash
php artisan make:migration create_posts_table
```

**Что делает эта команда:**

- Создаёт файл в директории `database/migrations/`
- Имя файла содержит timestamp для сохранения порядка выполнения
- Генерирует шаблон с методами `up()` и `down()`

Откройте созданный файл и заполните:

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('posts', function (Blueprint $table) {
            $table->id();
            $table->string('heading', 200);
            $table->text('body');

            // Внешний ключ на пользователя
            $table->foreignId('user_id')
                  ->constrained('users')
                  ->onDelete('cascade');

            $table->timestamps();

            // Индекс для ускорения сортировки и поиска по заголовку
            $table->index('heading', 'idx_heading');
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('posts');
    }
};
```

**Объяснение внешних ключей:**

- `foreignId('user_id')` — создаёт колонку `BIGINT UNSIGNED`
- `constrained('users')` — создаёт внешний ключ на таблицу users
- `onDelete('cascade')` — при удалении пользователя удаляются все его посты
- Альтернативы: `onDelete('set null')`, `onDelete('restrict')`

### Шаг 2.4. Создание миграции для таблицы comments

```bash
php artisan make:migration create_comments_table
```

Заполните миграцию:

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('comments', function (Blueprint $table) {
            $table->id();
            $table->text('body');

            // Внешний ключ на пост
            $table->foreignId('post_id')
                  ->constrained('posts')
                  ->onDelete('cascade');

            // Внешний ключ на пользователя
            $table->foreignId('user_id')
                  ->constrained('users')
                  ->onDelete('cascade');

            // Самоссылающийся внешний ключ для вложенных комментариев
            $table->foreignId('parent_id')
                  ->nullable()
                  ->constrained('comments')
                  ->onDelete('cascade');

            $table->timestamps();

            // Индексы для оптимизации запросов
            $table->index('post_id', 'idx_post_id');
            $table->index('parent_id', 'idx_parent_id');
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('comments');
    }
};
```

**Важная концепция — самоссылающиеся связи:**

- `parent_id` указывает на другой комментарий в той же таблице
- Позволяет создавать древовидные структуры (ответы на комментарии)
- `nullable()` означает, что корневые комментарии не имеют родителя

### Шаг 2.5. Создание миграции для таблицы likes (полиморфные отношения)

```bash
php artisan make:migration create_likes_table
```

Заполните:

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('likes', function (Blueprint $table) {
            $table->id();

            $table->foreignId('user_id')
                  ->constrained('users')
                  ->onDelete('cascade');

            // Полиморфные отношения:
            // likeable_type хранит имя модели: 'App\Models\Post' или 'App\Models\Comment'
            // likeable_id   хранит ID конкретной записи
            $table->string('likeable_type', 50);
            $table->unsignedBigInteger('likeable_id');

            $table->timestamp('created_at')->useCurrent();

            // Уникальный составной индекс:
            // один пользователь может лайкнуть объект только один раз
            $table->unique(
                ['user_id', 'likeable_type', 'likeable_id'],
                'idx_unique_like'
            );

            // Индекс для поиска всех лайков конкретного объекта
            $table->index(
                ['likeable_type', 'likeable_id'],
                'idx_likeable_type_id'
            );
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('likes');
    }
};
```

**Полиморфные отношения — мощная концепция Laravel:**

- Одна таблица может хранить связи с разными моделями
- `likeable_type` хранит имя класса модели
- `likeable_id` хранит ID записи
- Избегаем дублирования таблиц (`likes_for_posts`, `likes_for_comments`)

## Часть 3. Модели Eloquent ORM — объектно-реляционное отображение

### Шаг 3.1. Что такое Eloquent ORM

**Eloquent** — это реализация паттерна ActiveRecord в Laravel. Каждая
таблица БД имеет соответствующую модель.

**Преимущества Eloquent:**

- Читаемый, выразительный синтаксис
- Автоматическая защита от SQL-инъекций
- Удобная работа со связями между таблицами
- Автоматическое управление timestamps
- Поддержка аксессоров и мутаторов

### Шаг 3.2. Модель User

Модель User уже существует в `app/Models/User.php`. Модифицируем её:

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Illuminate\Database\Eloquent\Relations\HasMany;

class User extends Authenticatable
{
    use HasFactory, Notifiable;

    /**
     * Массово заполняемые атрибуты.
     * Защита от массового заполнения (Mass Assignment).
     */
    protected $fillable = [
        'lastname',
        'firstname',
        'patronymic',
        'email',
        'password',
    ];

    /**
     * Скрытые атрибуты (не включаются в JSON-ответы).
     */
    protected $hidden = [
        'password',
        'remember_token',
    ];

    /**
     * Приведение типов атрибутов.
     */
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password' => 'hashed', // Автоматическое хеширование при записи
        ];
    }

    /**
     * Отношение: пользователь имеет много постов.
     */
    public function posts(): HasMany
    {
        return $this->hasMany(Post::class);
    }

    /**
     * Отношение: пользователь имеет много комментариев.
     */
    public function comments(): HasMany
    {
        return $this->hasMany(Comment::class);
    }

    /**
     * Отношение: пользователь имеет много лайков.
     */
    public function likes(): HasMany
    {
        return $this->hasMany(Like::class);
    }

    /**
     * Аксессор: полное имя.
     * Использование: $user->full_name
     */
    public function getFullNameAttribute(): string
    {
        return trim("{$this->lastname} {$this->firstname} {$this->patronymic}");
    }

    /**
     * Аксессор: инициалы (Иванов И.И.)
     * Использование: $user->initials
     */
    public function getInitialsAttribute(): string
    {
        $patronymic = $this->patronymic
            ? ' ' . mb_substr($this->patronymic, 0, 1) . '.'
            : '';

        return $this->lastname . ' '
             . mb_substr($this->firstname, 0, 1) . '.'
             . $patronymic;
    }
}
```

**Объяснение ключевых концепций:**

1. **Mass Assignment Protection:**
   - `$fillable` — белый список полей, разрешённых для массового заполнения
   - Защищает от атак, когда злоумышленник добавляет лишние поля в запрос

2. **Аксессоры (Accessors):**
   - Методы вида `getXxxAttribute()` создают вычисляемые атрибуты
   - Атрибуты не хранятся в БД, а вычисляются на лету
   - Доступ через `snake_case`: `$user->full_name`

3. **Приведение типов (Casting):**
   - `hashed` — автоматически хеширует пароль при записи в БД

### Шаг 3.3. Создание модели Post

```bash
php artisan make:model Post
```

Откройте `app/Models/Post.php`:

```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;
use Illuminate\Database\Eloquent\Relations\MorphMany;

class Post extends Model
{
    use HasFactory;

    protected $fillable = [
        'heading',
        'body',
        'user_id',
    ];

    /**
     * Пост принадлежит пользователю.
     * Обратная сторона связи hasMany.
     */
    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }

    /**
     * Пост имеет много комментариев.
     */
    public function comments(): HasMany
    {
        return $this->hasMany(Comment::class);
    }

    /**
     * Полиморфное отношение: пост может иметь много лайков.
     * morphMany автоматически работает с полями likeable_type и likeable_id.
     */
    public function likes(): MorphMany
    {
        return $this->morphMany(Like::class, 'likeable');
    }

    /**
     * Только корневые комментарии (без parent_id) с жадной загрузкой ответов.
     */
    public function rootComments(): HasMany
    {
        return $this->hasMany(Comment::class)
                    ->whereNull('parent_id')
                    ->with('replies');
    }

    /**
     * Количество лайков.
     */
    public function likesCount(): int
    {
        return $this->likes()->count();
    }

    /**
     * Проверка: лайкнул ли пользователь этот пост.
     */
    public function isLikedByUser(?User $user = null): bool
    {
        if (!$user) {
            return false;
        }

        return $this->likes()
                    ->where('user_id', $user->id)
                    ->exists();
    }

    /**
     * Аксессор: краткое описание (первые 150 символов без HTML-тегов).
     * Использование: $post->excerpt
     */
    public function getExcerptAttribute(): string
    {
        return mb_substr(strip_tags($this->body), 0, 150) . '...';
    }
}
```

**Типы отношений Eloquent:**

- `hasMany` — один ко многим (пост имеет много комментариев)
- `belongsTo` — обратная связь (пост принадлежит пользователю)
- `morphMany` — полиморфное отношение один ко многим

### Шаг 3.4. Создание модели Comment


```php
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;
use Illuminate\Database\Eloquent\Relations\MorphMany;

class Comment extends Model
{
    use HasFactory;

    protected $fillable = [
        'body',
        'post_id',
        'user_id',
        'parent_id',
    ];

    /**
     * Комментарий принадлежит посту.
     */
    public function post(): BelongsTo
    {
        return $this->belongsTo(Post::class);
    }

    /**
     * Комментарий принадлежит пользователю.
     */
    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }

    /**
     * Родительский комментарий (для вложенных).
     * Второй аргумент — название внешнего ключа (не 'comment_id', а 'parent_id').
     */
    public function parent(): BelongsTo
    {
        return $this->belongsTo(Comment::class, 'parent_id');
    }

    /**
     * Дочерние комментарии (ответы на этот комментарий).
     */
    public function replies(): HasMany
    {
        return $this->hasMany(Comment::class, 'parent_id');
    }

    /**
     * Полиморфное отношение: лайки комментария.
     */
    public function likes(): MorphMany
    {
        return $this->morphMany(Like::class, 'likeable');
    }

    /**
     * Количество лайков.
     */
    public function likesCount(): int
    {
        return $this->likes()->count();
    }

    /**
     * Проверка лайка пользователем.
     */
    public function isLikedByUser(?User $user = null): bool
    {
        if (!$user) {
            return false;
        }

        return $this->likes()
                    ->where('user_id', $user->id)
                    ->exists();
    }

    /**
     * Является ли комментарий корневым (не ответом).
     */
    public function isRoot(): bool
    {
        return is_null($this->parent_id);
    }
}
```

### Шаг 4.1. Зачем нужны Form Requests

**Form Request** — специальный класс для валидации входящих данных.

**Преимущества:**

1. **Разделение ответственности** — контроллер не загромождён логикой валидации

3. **Авторизация на уровне запроса** — проверка прав до выполнения контроллера
4. **Автоматическая обработка ошибок** — редирект с ошибками и сохранение значений форм


```php
<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class StoreCommentRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true; // auth middleware в маршруте уже проверил авторизацию
    }

    public function rules(): array
    {
        return [
            'body' => ['required', 'string', 'min:2', 'max:5000'],

            'post_id' => [
                'required',
                'integer',
                'exists:posts,id', // Пост должен существовать в базе
            ],

            'parent_id' => [
                'nullable',
                'integer',
                // Родительский комментарий должен существовать
                // И обязательно принадлежать тому же посту
                Rule::exists('comments', 'id')->where(function ($query) {
                    $query->where('post_id', $this->input('post_id'));
                }),
            ],
        ];
    }

    public function messages(): array
    {
        return [
            'body.required'      => 'Комментарий не может быть пустым',
            'body.min'           => 'Комментарий слишком короткий',
            'post_id.exists'     => 'Пост не найден',
            'parent_id.exists'   => 'Родительский комментарий не найден или принадлежит другому посту',
        ];
    }
}
```

> **Почему убрали двойной `exists`:** В предыдущей версии было два
> отдельных правила `exists` для `parent_id`: одно простое и одно с
> дополнительным условием. Это дублирование — более специфичное правило
> `Rule::exists()->where(...)` уже включает в себя проверку
> существования записи. Оставляем только его.

## Часть 5. Слоистая архитектура и бизнес-логика (SOLID)

### Шаг 5.1. Принципы SOLID и слоистая архитектура

**SOLID — пять принципов объектно-ориентированного программирования:**

1. **S — Single Responsibility** — каждый класс делает одно дело
2. **O — Open/Closed** — классы открыты для расширения, закрыты для изменения
3. **L — Liskov Substitution** — наследники должны быть взаимозаменяемы
4. **I — Interface Segregation** — много специализированных интерфейсов лучше одного большого
5. **D — Dependency Inversion** — зависимость от абстракций, не от конкретных реализаций

### Шаг 5.2. Создание сервисного слоя

Создайте директорию вручную: `app/Services`

**Сервис для работы с постами** — `app/Services/PostService.php`:

```php
<?php

namespace App\Services;

use App\Models\Post;
use App\Models\User;
use Illuminate\Pagination\LengthAwarePaginator;
use Illuminate\Support\Facades\DB;

class PostService
{
    /**
     * Получить все посты с пагинацией.
     * with() — жадная загрузка связей, чтобы не делать N+1 запросов.
     * withCount() — добавляет атрибуты comments_count и likes_count.
     */
    public function getAllPosts(int $perPage = 10): LengthAwarePaginator
    {
        return Post::with(['user', 'likes'])
                   ->withCount(['comments', 'likes'])
                   ->latest()
                   ->paginate($perPage);
    }

    /**
     * Получить пост со всеми связями для страницы просмотра.
     */
    public function getPostWithRelations(int $postId): ?Post
    {
        return Post::with([
            'user',
            'rootComments.user',
            'rootComments.replies.user',
            'rootComments.likes',
            'rootComments.replies.likes',
        ])
        ->withCount(['comments', 'likes'])
        ->find($postId);
    }

    /**
     * Создать новый пост.
     * DB::transaction() гарантирует атомарность: либо всё сохранится, либо ничего.
     */
    public function createPost(array $data, User $user): Post
    {
        return DB::transaction(function () use ($data, $user) {
            $post = new Post();
            $post->heading = $data['heading'];
            $post->body    = $data['body'];
            $post->user_id = $user->id;
            $post->save();

            return $post;
        });
    }

    /**
     * Обновить пост.
     */
    public function updatePost(Post $post, array $data): Post
    {
        return DB::transaction(function () use ($post, $data) {
            $post->heading = $data['heading'];
            $post->body    = $data['body'];
            $post->save();

            return $post;
        });
    }

    /**
     * Удалить пост.
     * Комментарии и лайки удалятся автоматически благодаря CASCADE в БД.
     */
    public function deletePost(Post $post): bool
    {
        return DB::transaction(fn () => $post->delete());
    }

    /**
     * Получить посты конкретного пользователя.
     */
    public function getUserPosts(User $user, int $perPage = 10): LengthAwarePaginator
    {
        return $user->posts()
                    ->withCount(['comments', 'likes'])
                    ->latest()
                    ->paginate($perPage);
    }
}
```

**Сервис для лайков** — `app/Services/LikeService.php`:

```php
<?php

namespace App\Services;

use App\Models\Like;
use App\Models\User;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Support\Facades\DB;

class LikeService
{
    /**
     * Переключить лайк: если есть — убрать, если нет — поставить.
     * Возвращает массив с действием ('added'/'removed') и новым количеством лайков.
     */
    public function toggleLike(Model $likeable, User $user): array
    {
        return DB::transaction(function () use ($likeable, $user) {
            $like = Like::where('user_id', $user->id)
                        ->where('likeable_type', get_class($likeable))
                        ->where('likeable_id', $likeable->id)
                        ->first();

            if ($like) {
                $like->delete();
                return [
                    'action'      => 'removed',
                    'likes_count' => $likeable->likes()->count(),
                ];
            }

            Like::create([
                'user_id'       => $user->id,
                'likeable_type' => get_class($likeable),
                'likeable_id'   => $likeable->id,
            ]);

            return [
                'action'      => 'added',
                'likes_count' => $likeable->likes()->count(),
            ];
        });
    }
}
```

```php
    public function getPostCommentsTree(Post $post)
```

## Часть 6. Контроллеры — управление HTTP-запросами

> **Важно:** В Laravel 11 и 12 метод `$this->middleware()` в конструкторе
> контроллера **удалён**. Весь middleware теперь задаётся в маршрутах
> (файл `routes/web.php`). Контроллеры ниже написаны без middleware в
> конструкторе — правильная настройка будет в Части 7.

> **О методе `$this->authorize()`:** В контроллерах используется метод
> `$this->authorize('update', $post)`. Он опирается на **Policy** —
> класс с правилами доступа. PostPolicy и CommentPolicy мы создадим в
> Части 12, но можете сразу перейти туда, если видите ошибку
> `AuthorizationException`.

### Шаг 6.1. Контроллер для постов

```bash
php artisan make:controller PostController --resource
```

Откройте `app/Http/Controllers/PostController.php`:

```php
<?php

namespace App\Http\Controllers;

use App\Http\Requests\StorePostRequest;
use App\Http\Requests\UpdatePostRequest;
use App\Models\Post;
use App\Services\PostService;
use Illuminate\Http\RedirectResponse;
use Illuminate\View\View;

class PostController extends Controller
{
    /**
     * Внедрение зависимости через конструктор.
     * Laravel автоматически создаёт PostService и передаёт сюда.
     * Нам не нужно писать new PostService() — фреймворк делает это сам.
     */
    public function __construct(
        private PostService $postService
    ) {}

    /**
     * Список всех постов.
     */
    public function index(): View
    {
        $posts = $this->postService->getAllPosts(perPage: 15);

        return view('posts.index', compact('posts'));
    }

    /**
     * Форма создания поста.
     */
    public function create(): View
    {
        return view('posts.create');
    }

    /**
     * Сохранение нового поста.
     * StorePostRequest автоматически валидирует данные до входа в этот метод.
     */
    public function store(StorePostRequest $request): RedirectResponse
    {
        $post = $this->postService->createPost(
            $request->validated(), // только прошедшие валидацию поля
            $request->user()
        );

        return redirect()
            ->route('posts.show', $post)
            ->with('success', 'Пост успешно создан!');
    }

    /**
     * Просмотр одного поста.
     * Route Model Binding: Laravel автоматически найдёт Post по ID из URL.
     */
    public function show(Post $post): View
    {
        $post = $this->postService->getPostWithRelations($post->id);

        return view('posts.show', compact('post'));
    }

    /**
     * Форма редактирования поста.
     */
    public function edit(Post $post): View
    {
        // authorize() проверит PostPolicy::update() — см. Часть 12
        $this->authorize('update', $post);

        return view('posts.edit', compact('post'));
    }

    /**
     * Обновление поста.
     * Авторизация проверяется в UpdatePostRequest::authorize().
     */
    public function update(UpdatePostRequest $request, Post $post): RedirectResponse
    {
        $post = $this->postService->updatePost($post, $request->validated());

        return redirect()
            ->route('posts.show', $post)
            ->with('success', 'Пост успешно обновлён!');
    }

    /**
     * Удаление поста.
     */
    public function destroy(Post $post): RedirectResponse
    {
        $this->authorize('delete', $post);

        $this->postService->deletePost($post);

        return redirect()
            ->route('posts.index')
            ->with('success', 'Пост успешно удалён!');
    }
}
```

**Ключевые концепции:**

1. **Dependency Injection:** `PostService` внедряется автоматически через
   конструктор — Laravel сам создаёт объект через Service Container.
   Нам не нужно писать `new PostService()`.

2. **Route Model Binding:** `Post $post` в параметрах метода — Laravel
   автоматически ищет пост с ID из URL и передаёт готовый объект.
   Если не найдёт — вернёт 404.

3. **`$request->validated()`:** Возвращает только поля, прошедшие
   валидацию. Это безопаснее, чем `$request->all()`.

## Часть 7. Маршрутизация (Routing)

### Шаг 7.1. Определение маршрутов

> **Почему middleware теперь в маршрутах?** В Laravel 11 и 12 контроллеры
> стали «тонкими» — они не знают о middleware. Всё управление доступом
> к URL-адресам описывается в одном месте: `routes/web.php`. Это делает
> архитектуру предсказуемой: заглянув в файл маршрутов, сразу видно,
> какие адреса защищены.

Откройте `routes/web.php` и замените всё содержимое:

```php
<?php

use App\Http\Controllers\PostController;
use App\Http\Controllers\CommentController;
use App\Http\Controllers\LikeController;
use App\Http\Controllers\ProfileController;
use Illuminate\Support\Facades\Route;

/*
|--------------------------------------------------------------------------
| Публичные маршруты (доступны всем)
|--------------------------------------------------------------------------
*/

// Главная страница — список постов
Route::get('/', [PostController::class, 'index'])->name('home');

// Просмотр списка постов и одного поста — доступны без авторизации
Route::resource('posts', PostController::class)->only(['index', 'show']);

/*
|--------------------------------------------------------------------------
| Защищённые маршруты (только для авторизованных пользователей)
|--------------------------------------------------------------------------
| middleware('auth') автоматически перенаправит на страницу входа,
| если пользователь не авторизован.
*/

Route::middleware('auth')->group(function () {

    // Посты: создание, редактирование, удаление
    Route::resource('posts', PostController::class)
         ->except(['index', 'show']);

    // Комментарии
    Route::post('/comments', [CommentController::class, 'store'])
         ->name('comments.store');

    Route::delete('/comments/{comment}', [CommentController::class, 'destroy'])
         ->name('comments.destroy');

    // Лайки (AJAX-запросы, возвращают JSON)
    Route::post('/posts/{post}/like', [LikeController::class, 'togglePostLike'])
         ->name('posts.like');

    Route::post('/comments/{comment}/like', [LikeController::class, 'toggleCommentLike'])
         ->name('comments.like');

    // Профиль
    Route::get('/profile', [ProfileController::class, 'show'])
         ->name('profile.show');
});
```

**Объяснение концепций:**

1. **`Route::resource()`** создаёт 7 стандартных маршрутов:
   - `GET /posts` — `index`
   - `GET /posts/create` — `create`
   - `POST /posts` — `store`
   - `GET /posts/{post}` — `show`
   - `GET /posts/{post}/edit` — `edit`
   - `PUT /posts/{post}` — `update`
   - `DELETE /posts/{post}` — `destroy`

2. **`->only([...])`** и **`->except([...])`** — ограничивают набор
   создаваемых маршрутов. Мы разделили публичные и защищённые действия.

3. **`Route::middleware('auth')->group()`** — применяет middleware
   `auth` ко всем маршрутам внутри группы.

4. **Именованные маршруты** (`->name('posts.show')`) — позволяют
   генерировать URL в коде: `route('posts.show', $post)`. Если URL
   изменится, менять нужно только в одном месте.

## Часть 8. Middleware — промежуточное ПО

### Шаг 8.1. Что такое Middleware

**Middleware** — это фильтры HTTP-запросов. Выполняются до или после
обработки запроса контроллером.

**Типичные применения:**

- Аутентификация
- CORS (кросс-доменные запросы)
- Логирование запросов
- Ограничение частоты запросов (Rate Limiting)

**Встроенные middleware Laravel:**

- `auth` — требует аутентификации
- `guest` — только для гостей (незарегистрированных)
- `throttle` — ограничение частоты запросов
- `verified` — требует подтверждения email

### Шаг 8.2. Создание кастомного Middleware

Создадим middleware для логирования действий пользователя:

```bash
php artisan make:middleware LogUserActivity
```

Откройте `app/Http/Middleware/LogUserActivity.php`:

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Log;
use Symfony\Component\HttpFoundation\Response;

class LogUserActivity
{
    /**
     * Обработка входящего запроса.
     *
     * Метод handle() получает запрос и «следующий» обработчик ($next).
     * Вызов $next($request) передаёт запрос дальше по цепочке.
     * Код ДО $next выполняется до контроллера, ПОСЛЕ $next — после.
     */
    public function handle(Request $request, Closure $next): Response
    {
        // Код ДО выполнения контроллера
        if (auth()->check()) {
            Log::info('User activity', [
                'user_id' => auth()->id(),
                'email'   => auth()->user()->email,
                'url'     => $request->fullUrl(),
                'method'  => $request->method(),
                'ip'      => $request->ip(),
            ]);
        }

        // Передаём запрос дальше
        $response = $next($request);

        // Код ПОСЛЕ выполнения контроллера (здесь можно модифицировать ответ)

        return $response;
    }
}
```

**Регистрация middleware** в `bootstrap/app.php`:

```php
->withMiddleware(function (Middleware $middleware) {
    $middleware->alias([
        'log.activity' => \App\Http\Middleware\LogUserActivity::class,
    ]);
})
```

**Использование в маршрутах:**

```php
Route::get('/posts', [PostController::class, 'index'])
     ->middleware('log.activity');
```

### Шаг 8.3. Middleware для проверки владельца поста


```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Symfony\Component\HttpFoundation\Response;

class CheckPostOwner
{
    public function handle(Request $request, Closure $next): Response
    {
        $post = $request->route('post');

        if ($post && $post->user_id !== auth()->id()) {
            abort(403, 'У вас нет доступа к этому посту');
        }

        return $next($request);
    }
}
```

Создадим команду для генерации тестовых данных:

```bash

```

Откройте `app/Console/Commands/GenerateTestData.php`:

```php
<?php

namespace App\Console\Commands;

use App\Models\User;
use App\Models\Post;
use App\Models\Comment;
use Illuminate\Console\Command;
use Illuminate\Support\Facades\Hash;

class GenerateTestData extends Command
{
    /**
     * Сигнатура: app:generate-test-data — имя команды.
     * {users=10}    — обязательный аргумент (пользователи), по умолчанию 10.
     * {--posts=5}   — опциональный флаг (посты на пользователя), по умолчанию 5.
     * {--comments=3}— опциональный флаг (комментарии на пост), по умолчанию 3.
     */
    protected $signature = 'app:generate-test-data
                            {users=10 : Количество пользователей}
                            {--posts=5 : Количество постов на пользователя}
                            {--comments=3 : Количество комментариев на пост}';

    protected $description = 'Генерация тестовых данных для блога';

    public function handle(): int
    {
        $this->info('Начинаем генерацию тестовых данных...');

        $usersCount      = (int) $this->argument('users');
        $postsPerUser    = (int) $this->option('posts');
        $commentsPerPost = (int) $this->option('comments');

        $bar = $this->output->createProgressBar($usersCount);
        $bar->start();

        for ($i = 1; $i <= $usersCount; $i++) {
            $user = User::create([
                'lastname'   => "Фамилия{$i}",
                'firstname'  => "Имя{$i}",
                'patronymic' => "Отчество{$i}",
                'email'      => "user{$i}@example.com",
                'password'   => Hash::make('password'),
            ]);

            for ($j = 1; $j <= $postsPerUser; $j++) {
                $post = Post::create([
                    'heading' => "Пост #{$j} автора {$user->full_name}",
                    'body'    => "Содержание поста #{$j}. Lorem ipsum dolor sit amet, consectetur adipiscing elit.",
                    'user_id' => $user->id,
                ]);

                for ($k = 1; $k <= $commentsPerPost; $k++) {
                    Comment::create([
                        'body'      => "Комментарий #{$k} к посту #{$post->id}",
                        'post_id'   => $post->id,
                        'user_id'   => User::inRandomOrder()->value('id') ?? $user->id,
                        'parent_id' => null,
                    ]);
                }
            }

            $bar->advance();
        }

        $bar->finish();
        $this->newLine();
        $this->info('✓ Данные успешно сгенерированы!');

        $this->table(
            ['Тип', 'Количество'],
            [
                ['Пользователи', $usersCount],
                ['Посты', $usersCount * $postsPerUser],
                ['Комментарии', $usersCount * $postsPerUser * $commentsPerPost],
            ]
        );

        return Command::SUCCESS;
    }
}
```

**Запуск команды:**



### Шаг 11.9. JavaScript для лайков

Создайте файл `resources/js/likes.js`:

```javascript
/**
 * Обработчик лайков для постов и комментариев.
 * Работает через Fetch API (AJAX) — страница не перезагружается.
 */
document.addEventListener('DOMContentLoaded', function () {

    // Получаем CSRF-токен из meta-тега (добавлен в layouts/app.blade.php)
    const csrfToken = document.querySelector('meta[name="csrf-token"]').content;

    /**
     * Общая функция для переключения лайка.
     * @param {HTMLElement} button - кнопка лайка
     * @param {string} selector   - CSS-класс счётчика внутри кнопки
     */
    async function toggleLike(button, selector) {
        const url = button.dataset.url;
        const countSpan = button.querySelector(selector);

        try {
            const response = await fetch(url, {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json',
                    'X-CSRF-TOKEN': csrfToken,
                    'Accept': 'application/json',
                },
            });

            // Если не авторизован — перенаправляем на страницу входа
            if (response.status === 401) {
                window.location.href = '/login';
                return;
            }

            const data = await response.json();

            if (data.success) {
                // Обновляем счётчик
                countSpan.textContent = data.likes_count;

                // Переключаем визуальное состояние кнопки
                if (data.action === 'added') {
                    button.classList.add('liked');
                } else {
                    button.classList.remove('liked');
                }
            }
        } catch (error) {
            console.error('Ошибка при обработке лайка:', error);
        }
    }

    // Лайки на постах
    document.querySelectorAll('.like-btn').forEach(button => {
        button.addEventListener('click', function () {
            toggleLike(this, '.likes-count');
        });
    });

    // Лайки на комментариях
    document.querySelectorAll('.like-btn-comment').forEach(button => {
        button.addEventListener('click', function () {
            toggleLike(this, 'span');
        });
    });
});
```

> **Почему файл в `resources/js/`, а не в `public/js/`?**
> Vite следит за файлами в `resources/` и обрабатывает их при сборке
> (минификация, проверка совместимости). Файлы из `public/` не
> обрабатываются. Подключаем через `@vite('resources/js/likes.js')`.

## Часть 12. Политики доступа (Policies)

### Шаг 12.1. Создание PostPolicy

```bash
php artisan make:policy PostPolicy --model=Post
php artisan make:policy CommentPolicy --model=Comment
```

Откройте `app/Policies/PostPolicy.php`:

```php
<?php

namespace App\Policies;

use App\Models\Post;
use App\Models\User;

class PostPolicy
{
    /**
     * Все могут просматривать посты.
     * Знак ? перед User означает, что пользователь может быть не авторизован.
     */
    public function view(?User $user, Post $post): bool
    {
        return true;
    }

    /**
     * Только авторизованные могут создавать посты.
     */
    public function create(User $user): bool
    {
        return true;
    }

    /**
     * Редактировать пост может только его автор.
     */
    public function update(User $user, Post $post): bool
    {
        return $user->id === $post->user_id;
    }

    /**
     * Удалять пост может только его автор.
     */
    public function delete(User $user, Post $post): bool
    {
        return $user->id === $post->user_id;
    }
}
```

Откройте `app/Policies/CommentPolicy.php`:

```php
<?php

namespace App\Policies;

use App\Models\Comment;
use App\Models\User;

class CommentPolicy
{
    /**
     * Удалять комментарий может только его автор.
     */
    public function delete(User $user, Comment $comment): bool
    {
        return $user->id === $comment->user_id;
    }
}
```

> **Регистрация Policy:** В Laravel 11 и 12 политики **обнаруживаются
> автоматически** по соглашению об именах: `PostPolicy` для модели
> `Post`, `CommentPolicy` для `Comment`. Ничего дополнительно
> регистрировать не нужно.

**Использование в контроллере:**

```php
$this->authorize('update', $post); // вызовет PostPolicy::update()
$this->authorize('delete', $comment); // вызовет CommentPolicy::delete()
```

**Использование в Blade:**

```blade
@can('update', $post)
    <a href="{{ route('posts.edit', $post) }}">Редактировать</a>
@endcan

@can('delete', $comment)
    {{-- кнопка удаления --}}
@endcan
```
